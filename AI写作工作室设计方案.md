## 📐 一、整体架构

### 1.1 功能定位

```
内容生产全流程覆盖：
想法/素材 → AI写作工作室 → 精品文章 → 多平台发布
     ↓            ↓            ↓           ↓
  用户输入    三位AI协作    编辑器呈现   一键分发
```

### 1.2 三个AI角色定义

| 角色 | 定位 | 核心能力 | AI模型建议 |
|------|------|----------|-----------|
| 🎩 **丧彪**（主编） | 流程总监 + 质量把关 | 风格分析、任务分配、稿件审核 | GPT-4 / Claude 3.5 |
| ✍️ **阿强**（写手） | 内容创作 + 风格适配 | 文章改写、风格模仿、内容优化 | GPT-4 Turbo |
| 📚 **小美**（助理） | 素材支撑 + 数据验证 | 案例搜索、数据核验、素材整理 | GPT-4 + 搜索API |

## ⚙️ 三、工作流程设计

### 3.1 标准工作流

```
用户输入
    ↓
【第1轮：风格分析与方案制定】
丧彪：分析原文/目标风格 → 制定改写方案
    ↓
【第2轮：内容创作】
阿强：按方案改写 → 标记需要素材的地方 → @小美
    ↓
【第3轮：素材补充】
小美：搜索案例 → 核验数据 → 反馈给阿强
    ↓
【第4轮：内容完善】
阿强：整合素材 → 完善文章 → 提交给丧彪
    ↓
【第5轮：质量审核】
丧彪：审核稿件 → 提出优化建议（99条）
    ↓
【第6轮：最终优化】
阿强：根据建议优化 → 生成最终版
    ↓
输出到编辑器
```

### 3.2 使用场景

#### 场景1：从零开始创作
```
输入：话题 + 关键词 + 目标风格
输出：完整的风格化文章
```

#### 场景2：改写现有文章
```
输入：原文 + 目标风格（如"刘润风格"）
输出：风格适配的新版本
```

#### 场景3：内容优化
```
输入：已有草稿 + 优化方向
输出：优化建议 + 改进版本
```
